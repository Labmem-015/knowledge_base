# Статический базовый анализ
* Анализ сигнатур при помощи антивируса
* Вычисление хэша (MD5, SHA-1)
* Поиск строк при помощи утилит (Strings из Sysinternals на WIndows, или strings на UNIX). Там можно задать критерии поиска (например длину строк от `n` символов). Можно искать как ASCII, так и Unicode последовательности (на Windows это wstring, т.е. UTF-16).

# Методы скрытого запуска вредоноса
> [!important] 
> Эти заметки нужны для понимания того, как работает скрытый запуск вредоноса с целью противодействия данным мероприятиям. Это нужно для кибербезопасности. По вызовам специальных функций можно проанализировать потенциально вредоносное ПО и сделать вердикт
## Внедрение DLL
Вредоносы используют `VirtualAllocEx` для выделения памяти во внешнем процессе. Далее в эту память записывается строка с названием DLL с помощью `WriteProcessMemory`. И наконец во внешнем процессе вызывается функция `LoadLibrary` из Kernel32.dll с передачей ей строки с называнием DLL из адресного пространства процесса. Чтобы `LoadLibray` вызывался от имени процесса, используется `CreateRemoteThread`.
## Прямое внедрение
Очень похоже на внедрение DLL, только там внедряется программный код или код командной оболочки.
## Подмена процесса
В данном случае вредоносы перезаписывают адресное пространство процесса, тем самым маскируясь без рисков обрушить процесс в случае внедрения. Распространённым примером является маскировка под svchost.exe.
Как это работает. Создаётся приостановленный процесс, главный поток которого не запускается. Потом освобождается вся память, выделенная под него и выделяется новая, в которую затем записывают новый образ исполняемого файла с вредоносом.
Код использует `CreateProcess` с флагом `CREATE_SUSPEND`. Деаллокация производится функцией `ZwUnmapViewOfSection`, выделяется новая память с помощью `VirtualAllocEx`. `WriteProcessMemory` записывает заголовки и секции исполняемого образа. В конце восстанавливается среда атакуемого процесса и происходит связывание точки входа с вредоносным кодом при помощи `SetThreadContext`. `ResumeThread` запускает поток с подменённым кодом.

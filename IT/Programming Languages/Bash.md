# Виды оболочек
## Profile shell
Входящая оболочка - это первая оболочка, которую пользователь получает после успешного входа в систему (через SSH, физический терминал, консоль). Она инициализирует сессию пользователя.
Обычно это файл `/etc/profile` или `~/.bash_profile`.
## Interactive shell
Интерактивная оболочка ждет ввода команд от пользователя и выводит приглашение (prompt).
Файл: `~/.bashrc` или `/etc/bash.bashrc`
## Noniteractive shell
Оболочка, которая просто выполняет скрипт. Она не является интерактивной, не подключает интерактивные конфиги, однако использует переменную окружения `BASH_ENV`.
## Частые проблемы с SSH
Самая частая проблема новичков: добавление экспорта переменной в `~/.bashrc`. Однако она не работает при входе по SSH. Это происходит потому, что SSH-сессия — это **login shell**, которая считывает `~/.bash_profile`, а не `~/.bashrc`.

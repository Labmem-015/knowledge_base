# Разделяемая память (shared memory)
Разделяемая память - один из методов повышения производительности при IPC. В отличие от RabbitMQ, Apache Kafka, Redis, она позволяет организовать обмен данными между процессами на одном хосте, в то время, как перечисленные технологии является примерами инструментов для организации межсетевого взаимодействия.
При открытии/создании объекта разделяемой памяти, он помещается в tempfs (Linux) - виртуальную файловую систему, которая находится не на физическом диске, а в RAM. Обычно располагается в `/dev/shm` Он не будет освобождать память, пока хотя бы один процесс использует его. Далее можно отобразить в локальное адресное пространство процесса эту разделяемую память. В `boost` есть реализации механизма разделяемой памяти.
`shm_open` - для открытия.
`ftruncate` - для задания размера разделяемой памяти.
`mmap` - можно использовать в качестве разделяемой памяти, но в отличие от `shm_open` время жизни такой памяти будет ограничиваться в рамках процесса.
# Сокеты
Сокет в программе представлен дескриптором на объект передачи данных (например файл в Unix-системе). Операция `close` закрывает дескриптор, так что его можно переиспользовать для открытия нового сокета. `shutdown` ограничивает передачу данных (либо отправку, либо получение данных, либо оба варианта), но при этом дескриптор остаётся открытым и его нужно закрыть. Это нужно для мягкого способа остановить поток данных. В то время как `close` разрушает объект и освобождает ресурсы из памяти.

# Семафор
Есть два типа: семафор со счётчиком, который ограничивает количество процессов, которые могут иметь одновременный доступ к ресурсу; и бинарный семафор, который принимает состояния `занят`/`не занят`.
Отличие бинарного семафора от мьютекса:
* Мьютекс может быть разблокирован только потоком, который его заблокировал.
* Бинарный семафор может быть просигнализирован любым потоком или процессом.

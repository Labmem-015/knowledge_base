# Services
`systemd` позволяет запускать службы. Из особенностей:
* он делает это параллельно;
* службы не ждут начала инициализации служб-зависимостей, а создают сокет-файлы, которые позволят взаимодействовать с зависимостью по готовности.
* на данный момент одна из самых быстрых средств запуска и управления службами.
* запуск службы при изменении содержимого директории
* запуск службы при изменении списка `/dev`
`systemd` оперирует юнитами (модулями). Они подразделяются на несколько типов.
* служба
* цель (target). Является обобщающей сущностью, которая позволяет оперировать с несколькими службами в группе
* snapshot для создания и управления слепками `systemd`
`systemctl` управляет службами и мониторит их. Команды всем известны.
# KVM
KVM - представляет собой модули ядра. Изначально на своей заре представлял собой драйвер ядра Linux, который предоставлял доступ к расширению аппаратной виртуализации процессора intel архитектуры x86. Он создавал символьное устройство `/dev/kvm`. А запатченный QEMU открывал это устройство и передавал команды для управления KVM через API системных вызовов `ioctl`.

В состав KVM входит Memory Management Unit, который создаёт теневые страницы памяти, производит трансляцию адресов гостевых систем в физические адреса хоста.
Также в KVM входит эмулятор/декодер x86 инструкций, а также код для поддержки виртуализации процессора.

## KVM и VT-x
Есть стандартные уровни привилегий 0-3 у процессора. А есть расширение VMX (Virtual Machine Extension), которую предоставляет VT-x. В VMX есть два режима работы процессора:
* VMX root, который __называют__ уровнем `-1`
* VMX non-root, который соответствует уровням 0-3 на гостевой машине.
```
      Linux                     Linux/KVM Host                            Guests
        |                             |                                     |
+----------------+            +----------------+                    +---------------+
|                |    VMXON   |                |  VMLAUNCH/VMRESUME |               |  
| Protected mode |  --------> | VMX root mode  | -----------------> |  VMX non-root |
|                |            |                |                    |               |
|   Rings 0-3    |    VMXOF   |   Rings 0-3    |      VM-exit       |   Rings 0-3   |
|                |  <-------- |                |  <---------------- |               |
+----------------+            +----------------+                    +---------------+
```

# Services
`systemd` позволяет запускать службы. Из особенностей:
* он делает это параллельно;
* службы не ждут начала инициализации служб-зависимостей, а создают сокет-файлы, которые позволят взаимодействовать с зависимостью по готовности.
* на данный момент одна из самых быстрых средств запуска и управления службами.
* запуск службы при изменении содержимого директории
* зte1апуск службы при изменении списка `/dev`
`systemd` оперирует юнитами (модулями). Они подразделяются на несколько типов.
* служба
* цель (target). Является обобщающей сущностью, которая позволяет оперировать с несколькими службами в группе
* snapshot для создания и управления слепками `systemd`
`systemctl` управляет службами и мониторит их. Команды всем известны.
# KVM/QEMU
## Основные понятия
KVM - это гипервизор 1-го типа, встроенный в ядро Linux, работающий непосредственно с железом. Он занимается виртуализацией.
QEMU - это гипервизор 2-го типа, который работает поверх ОС и имеет возможность эмулировать оборудование и разные архитектуры.
Типы гипервизоров:
* 1-тип работает непосредственно на железе (KVM)
* 2-тип работает поверх операционной системы (QEMU)

## Как они работают
KVM состоит из нескольких компонентов:
* kvm.ko: это модуль ядра Linux, работает напрямую с железом и эмулирует память, процессор и виртуальные устройства для взаимодействия с хостом. Он использует аппаратные расширения для виртуализации типа Intel VT-x или AMD-V для обеспечения большей производительности.
* QEMU: может использоваться как менеджер виртуальных машин, а может использоваться как эмулятор железа, к которому обращается kvm для виртуализации, вместо Intel VT-x или AMD-V. Это открывает возможности для эмуляции других архитектур и запуска на них гостевых ОС.
* Менеджер виртуальных машин - контролирует весь жизненный цикл виртуалок (запуск, остановка, пауза, изменение конфигураций).
* Виртуальные устройства
* Гостевые ОС.
* libvirt - набор библиотек, инструментов и API для управления виртуализацией. Я предположу, что он используется в программировании для реализации собственных платформ/решений виртуализации типа CLI virsh, GUI решения Virtual Machine Manager, Gnome Boxes и прочие. Он поддерживает работу с гипервизорами KVM/QEMU, Hyper-V, VMWare Workstation, VirtualBox и другие.
* Паравиртуализация (KVM PV) - механизм, реализующий поддержку виртуализации без эмуляции. (Наверное имеется ввиду, что мы обращаемся не к виртуальному процу, а напрямую к гипервизору в обход виртуального проца?) Гостевая ОС в курсе, что она работает в виртуализированной среде и поэтому обращается напрямую к гипервизору через API для получения доступа к аппаратным ресурсам хоста. Но это требует адаптации гостевой системы к паравиртуализации через модификацию ядра или использование драйверов.
* Live Migration - миграция виртуалок между хостами без их прерывания.
* Управляющие инструменты (мониторинг, управление) типа Proxmox, oVirt, OpenStack и др.
## История KVM
KVM - представляет собой модули ядра. Изначально на своей заре представлял собой драйвер ядра Linux, который предоставлял доступ к расширению аппаратной виртуализации процессора intel архитектуры x86. Он создавал символьное устройство `/dev/kvm`. А запатченный QEMU открывал это устройство и передавал команды для управления KVM через API системных вызовов `ioctl`.

В состав KVM входит Memory Management Unit, который создаёт теневые страницы памяти, производит трансляцию адресов гостевых систем в физические адреса хоста.
Также в KVM входит эмулятор/декодер x86 инструкций, а также код для поддержки виртуализации процессора.

## KVM и VT-x
Есть стандартные уровни привилегий 0-3 у процессора. А есть расширение VMX (Virtual Machine Extension), которую предоставляет VT-x. В VMX есть два режима работы процессора:
* VMX root, который __называют__ уровнем `-1`
* VMX non-root, который соответствует уровням 0-3 на гостевой машине.
```
      Linux                     Linux/KVM Host                            Guests
        |                             |                                     |
+----------------+            +----------------+                    +---------------+
|                |    VMXON   |                |  VMLAUNCH/VMRESUME |               |  
| Protected mode |  --------> | VMX root mode  | -----------------> |  VMX non-root |
|                |            |                |                    |               |
|   Rings 0-3    |    VMXOF   |   Rings 0-3    |      VM-exit       |   Rings 0-3   |
|                |  <-------- |                |  <---------------- |               |
+----------------+            +----------------+                    +---------------+
```
